The Five Layers

1

Unified Intake (Micro-Frontend)

Hosted on Alchemy (Backstage)

A single, agentic intake experience using Google A2A/A2UI patterns. The AI parses natural language requests, categorizes them, and collects structured metadata through a conversational interface. Built with React and our existing component libraries.

React A2A/A2UI Backstage Plugin
2

AI Processing Layer

Categorization, Prioritization, Routing

AI Concierge applies rules, ML models, and value methodology to score and route requests. Includes automatic documentation generation and warm handoff to human reviewers when needed.

3

Governance Layer

Asset Inventory â€” This Project

The governance spine that ensures every asset is tracked, owned, and aligned to goals. Captures decision-oriented metadata: ownership, purpose, criticality, lifecycle stage, and 2026 T&O goal alignment. Validates intake requirements and prevents ungoverned assets.

4

Delivery Layer

Build, Deploy, Notify

AI agents assist with ETL optimization, dynamic report generation, and deployment automation. Jenkins CI/CD, GitHub Actions, and Webex notifications keep stakeholders informed.

5

Integration Fabric

Enterprise System Connectivity

APIs to ServiceNow (tickets), JIRA (stories), Confluence (documentation), Pathfinder/Clarity (portfolio management), and GitHub (code). The micro-frontend orchestrates these through a unified backend.
